Language Guide

Purpose

AIbasic is a natural-language programming language where you write instructions in plain English. The AI compiler translates your intent into optimized Python code. Focus on what you want to do, not how to do it.

Program Structure

<line_number> <instruction in natural language>
<line_number> (<task_type>) <instruction in natural language>

Basic Rules

  • Line numbers: Positive integers (10, 20, 30...)
  • Spacing: Use increments of 10 to allow insertions
  • Task types: Optional hints in parentheses: (csv), (postgres), (api)
  • Natural language: Write instructions as you would speak them

Examples

10 set x to 5
20 multiply x by 2 and store in result
30 if result is greater than 5 print "Large"
40 (csv) read file "data.csv" into dataset
50 (df) filter dataset where age > 18

Control Flow

Jumps (GOTO)

# Unconditional jump
30 goto 100

# Conditional jump
40 if x > 10 jump to line 100

Error Handling

10 on error goto 900
20 # risky operation
30 divide x by y

900 print "Error:" and _last_error
910 print "At line:" and _last_error_line

Error Variables:

  • _last_error — Exception object
  • _last_error_line — Line number of error
  • _error_handler — Current error handler line

Subroutines

10 set x to 5
20 call 1000
30 print "x is now" and x
40 goto 999

1000 print "In subroutine"
1010 multiply x by 2
1020 return

999 print "End"

Best Practice: Use line numbers 1000+, 2000+, etc. for subroutines

Variables

  • Declaration: Implicit, just use them
  • Assignment: set x to 5
  • Types: Inferred automatically (int, float, string, bool, list, dict, DataFrame)
  • Scope: Global by default

Examples

10 set counter to 0
20 set name to "Alice"
30 set prices to list 10.5, 20.0, 15.75
40 set user to dict with "name" as "Bob" and "age" as 30

Comments

# Full line comment
10 set x to 5  # Inline comment
20 print x

Pro Tips

  • Use task type hints for clarity: (postgres), (csv), (api)
  • Space line numbers by 10s for easy insertion
  • Use line 1000+ for subroutines, 9000+ for error handlers
  • Add comments to document complex logic
  • Test incrementally — compile and run frequently
