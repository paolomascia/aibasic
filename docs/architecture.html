Architecture — Executing Intent

Overview

The runtime is a distributed cognitive environment that executes intent graphs produced by the compiler. It selects strategies (data vs RPA vs API), provisions services, and orchestrates steps deterministically.

AI Context

  • symbol_table: variables, resources, agents
  • data_graph: lineage/derivations
  • capabilities: what actions are possible now
  • privacy/mutability: data governance and lifetimes

Data Fabric

Abstracts DB/files/streams as intelligent services. The runtime chooses engines, schemas, indices, and persistence transparently.

Communication Buses

Connectors for HTTP/REST, message queues, email, SSH. The programmer names the intent; the runtime handles protocols.

RPA Interfaces

Browser/desktop automation lives alongside data tasks. Selectors use robust textual/ARIA anchors; OCR normalizes output.

Capability Model

Maps an action (e.g., “make an Excel report”) to viable strategies (direct file writer vs RPA Excel) based on context, policy, and availability.

Self‑healing

Retries with exponential backoff, selector fallbacks, schema inference, and intent reinterpretation when safe.
